- **Dynamic Role-Based Access Control (RBAC):** Permissions are not hard-coded. They are fetched from the database and stored in the , allowing the UI to show/hide elements dynamically based on the current user's role.
- **Data Isolation:** Core principle where all major database queries for multi-department data (floors, employees, settings) are filtered by the user's  on the backend.
- **Debouncing:** Implemented on the National ID input field to provide real-time validation against the database without overwhelming the server with API calls on every keystroke.
- **Stateful UI Components:** The new dashboard and employee table components are designed to be highly interactive and stateful, reflecting data changes in real time.
- **Security Hardening:** Implemented account lockout after multiple failed login attempts, mandatory PIN change on first login, and session timeouts.
