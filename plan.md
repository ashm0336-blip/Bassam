### Phase 4 — P1: نظام إدارة المهام (بديل/تطوير المعاملات) ⏭️
> **المرحلة التالية المقترحة (قيد البدء)**

#### 4.1 Backend — Tasks API + DB Model
- إنشاء Collection جديدة: `tasks`.
- تصميم نموذج Task يدعم تعدد الموظفين:
  - `id`, `department`, `title`, `description`
  - `assignee_ids: [employee_id]` (Multi-assignee)
  - `created_by`, `created_at`, `due_at`, `priority`
  - `status: pending|in_progress|done|canceled|overdue`
  - `updates/comments` (اختياري)
- Endpoints مقترحة:
  - `GET /api/tasks?department=...` (مدير/أدمن يرى كل مهام الإدارة)
  - `GET /api/tasks/my` (الموظف يرى مهامه فقط)
  - `POST /api/tasks` (إنشاء مهمة متعددة المسندين)
  - `PUT /api/tasks/{id}` (تعديل المهمة)
  - `PUT /api/tasks/{id}/status` (تحديث الحالة من الموظف/المدير)
  - `POST /api/tasks/{id}/comment` (اختياري)
  - `DELETE /api/tasks/{id}` (صلاحية المدير/الأدمن)
- صلاحيات:
  - مدير الإدارة/الأدمن: create/update/delete + view all dept.
  - الموظف: view my + update status (حسب السياسة).

#### 4.2 Frontend — TasksPage.jsx (Kanban + List)
- إنشاء صفحة جديدة `TasksPage.jsx` بواجهتين:
  1) **Kanban**: أعمدة حسب الحالة مع Drag & Drop (اختياري).
  2) **List/Table**: مشابه لواجهة المعاملات الحالية (فلترة/بحث/تصدير لاحقاً).
- نموذج “إنشاء مهمة”:
  - قائمة منسدلة متعددة الاختيار للموظفين (تعكس موظفي الإدارة مباشرة).
  - حقل نص حر للعنوان/الوصف.
  - موعد/أولوية.
- عرض مهام الموظف:
  - الموظف يرى مهامه فقط مع زر تحديث الحالة.
- دمج مع نظام الإشعارات:
  - عند إنشاء/تعيين مهمة → إنشاء Notification + صوت.
  - عند تغيير الحالة/التعليق → إشعار للمدير (اختياري).

#### 4.3 استبدال المعاملات بالمهام عبر صفحات الإدارات
- استبدال استخدام `TransactionsPage` داخل:
  - `PlanningDepartment.jsx`
  - `CrowdServicesDepartment.jsx`
  - `MatafDepartment.jsx`
  - `GatesDepartment.jsx` / `PlazasDepartment.jsx` إن كانت تستخدمها
- تحديث Labels في التبويبات: “المعاملات” → “المهام اليومية”.

#### 4.4 Success Criteria (نظام المهام)
- مدير الإدارة:
  - يرى كل المهام في إدارته.
  - يرسل مهمة لموظف واحد أو عدة موظفين.
  - يتابع حالة التنفيذ.
- الموظف:
  - يرى مهامه فقط.
  - يحدّث الحالة بسهولة.
- الإشعارات:
  - جرس + صوت عند استلام مهمة.
- الاستقرار:
  - لا تعارض مع الجداول الشهرية والإحصائيات.
