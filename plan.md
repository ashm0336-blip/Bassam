# خطة تنفيذ مهام P0 لمنصة خدمات الحشود (V1) — **مُحدَّثة بعد الإكمال**

## 1) Objectives (الأهداف)
- ✅ ترقية **لوحة التحكم الرئيسية** لتصبح “غرفة عمليات” (KPIs حيّة + Heatmap + تنبيهات + Timeline + مؤشرات/رسوم).
- ✅ إضافة **Seed Data احترافي** لإحياء النظام (بوابات + موظفين + ساحات + مطاف + تنبيهات) مع إمكانية الحذف للبيانات التجريبية.
- ✅ بناء **واجهة ميدانية Mobile‑First** لإدخال الكثافات وإرسال التنبيهات بأقل نقرات.
- ✅ تفعيل **ملاحة (Sidebar) كاملة** مع أقسام فرعية للإدارات.
- 🎯 الهدف الحالي (ما بعد P0): تثبيت/تحسين (Stabilization) + بدء مهام P1 وفق أولوياتك.

## 2) Implementation Steps (خطوات التنفيذ)

### Phase 1 — Core POC (عزل المخاطر قبل البناء)
> **مكتملة بالكامل ✅**

**POC‑A: Seed Data Pipeline (Backend + DB)** ✅
- تم إنشاء Endpoint:
  - `POST /api/admin/seed/full` لتوليد بيانات احترافية
  - `DELETE /api/admin/seed/clear` لحذف بيانات التجربة فقط
- البيانات المُولَّدة (نتيجة فعلية):
  - ✅ **213** بوابة
  - ✅ **54** موظف + **55** حساب مستخدم
  - ✅ **12** ساحة
  - ✅ **3** مستويات مطاف
  - ✅ **6** تنبيهات
- ملاحظة: تم توفير تمييز للبيانات التجريبية عبر الحقل `_seed: true` لتسهيل الحذف.

**POC‑B: Dashboard “Ops Data” Endpoint** ✅
- تم إنشاء Endpoint مجمّع:
  - `GET /api/dashboard/ops`
- يعيد:
  - KPIs (بوابات، موظفين نشطين، نسبة إشغال، تنبيهات…)
  - Heatmap حسب الساحات (Utilization + open/total)
  - Smart alerts (قواعد بسيطة)
  - Recent alerts panel
  - Timeline من `activity_logs`
  - Overview للساحات

**POC‑C: Field Worker Minimal Flow** ✅
- تم إنشاء Endpoints:
  - `POST /api/field/density`
  - `POST /api/field/alert`
  - `GET /api/field/my-submissions`
- تم اختبار الإرسال عبر API بنجاح.

### Phase 2 — V1 App Development (تنفيذ واجهات P0)
> **مكتملة بالكامل ✅**

**2.1 Dashboard جديد (Frontend)** ✅
- تم تحديث صفحة `/` لتصبح “غرفة العمليات” وتتضمن:
  - KPI gauges (RadialBar)
  - Heatmap حسب الساحة
  - Smart Alerts panel مع CTA
  - رسوم توزيع الورديات (Bar)
  - Pie chart لتوزيع الموظفين حسب الإدارة
  - Recent alerts + Timeline
  - Department status cards
- مرتبط بالكامل مع `GET /api/dashboard/ops` مع تحديث تلقائي.

**User stories (Dashboard)** ✅
1) رؤية حالة البوابات في شاشة واحدة.
2) مؤشرات حيّة بدل أرقام جافة.
3) تنبيهات ذكية واضحة مع “ماذا أفعل الآن”.
4) Timeline للتسليم والاستلام.
5) تقليل الضوضاء عبر عرض مركّز (الساحات/الحالة).

**2.2 Seed Data (Backend + Admin Action)** ✅
- تم تجهيز seed عبر API (تشغيل + حذف بيانات تجريبية).
- جاهز للربط لاحقاً من واجهة الأدمن (إن رغبت بإضافة زر داخل UI كتحسين P1).

**2.3 Mobile‑First Field Worker UI** ✅
- تم إنشاء صفحة جديدة: `/field`
- تحتوي:
  - إدخال كثافة (4 مستويات: خفيف/متوسط/مزدحم/حرج)
  - إرسال تنبيه سريع (4 أنواع: كثافة/أمن/طبي/صيانة)
  - عرض آخر الإرسالات
- تصميم Mobile‑First (بطاقات كبيرة + أزرار واضحة + RTL).

**2.4 Compact Gate Display (200+ بوابة)** ✅ (جاهزية بيانات + دعم عرض سريع)
- تم توفير بيانات 200+ بوابة
- تم دعم عرض حالة البوابات حسب الساحة في الـ Dashboard (Heatmap)
- الخطوة التالية (تحسين P1 اختياري): صفحة مخصصة لعرض جدولي/مضغوط مع بحث/فلترة/تجميع داخل `/gates`.

**اختبار Phase 2** ✅
- تم التحقق عبر:
  - لقطات شاشة لصفحات: Dashboard / Field / Gates
  - اختبارات API لعمليات ops + field submit
- الخدمات تعمل وتعيد بيانات واقعية بعد seed.

### Phase 3 — Stabilization + Enhancements (بعد V1)
> **المرحلة التالية المقترحة (P1) ⏭️**

**3.1 تثبيت وتحسين الأداء**
- إضافة Indexes في Mongo للـ collections الأكثر استخداماً:
  - `gates.plaza`, `gates.status`, `alerts.is_read`, `activity_logs.timestamp`, `employees.department`
- كاش بسيط لـ `/api/dashboard/ops` (اختياري) لتقليل الضغط.

**3.2 تحسين Smart Alerts Rules**
- قواعد أكثر دقة (بوابة مفتوحة بلا موظف باستخدام mapping حقيقي، عتبات لكل ساحة، عقود + حالات حسابات مجمدة…)

**3.3 ربط seed بواجهة الأدمن (تحسين UX)**
- إضافة زر داخل Admin > Data Management:
  - تشغيل seed
  - حذف seed
  - عرض ملخص الأعداد

**3.4 تحسين العرض المضغوط للبوابات (P0.4 → تحسينات إضافية)**
- جدول مضغوط + بحث + فلاتر + تجميع حسب الساحات
- (اختياري) Virtualization لتحسين الأداء

**User stories (Stabilization)**
1) أداء ثابت حتى مع بيانات كثيرة.
2) Logs واضحة لأي أخطاء seed أو ops.
3) تنبيهات أدق تقلل الإنذارات الكاذبة.
4) تذكّر آخر اختيار للمستخدم الميداني.
5) صلاحيات دقيقة وحماية الوصول.
